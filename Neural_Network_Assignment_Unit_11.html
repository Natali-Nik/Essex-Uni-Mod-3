Unit 11: Neural Network Assignment

Training of a Convolutional Neural Network (CNN) in Python using the CIFAR-10 dataset for object recognition.

Key Lessons and Outcomes

This assignment was a steep learning curve, teaching me to balance simplicity and complexity in model architecture through iterative adjustments and optimizations. It improved my Python programming, debugging, and problem-solving skills while deepening my understanding of how Convolutional Neural Networks operate. I also gained a clear appreciation for the critical role of accurate data preprocessing, as errors in this step can significantly impact model performance.

Unit 11: Neural Network - Assignment Details

Assignment Details: The assignment focused on training a Convolutional Neural Network (CNN) using the CIFAR-10 dataset to evaluate its performance in object recognition across ten distinct categories. Key tasks included data preparation, designing and training the model, evaluating its accuracy, and deriving insights. The CNN was chosen due to its suitability for structured image data, emphasizing strategies, challenges, and learnings throughout the process.

Unit 11: From Data to Model Training

  • Data Preprocessing & Model Architecture: The workflow involved preparing the CIFAR-10 dataset by normalizing pixel values, splitting data, and one-hot encoding labels. The neural network included three convolutional layers with ReLU activation, batch normalization, max-pooling, and dropout, followed by a dense layer and a softmax output. Trained using the Adam optimizer and categorical cross-entropy loss, accuracy tracking monitored performance. Visualization tools provided insights into labels, images, and the model’s structure, enabling effective feature extraction, reduced overfitting, and accurate predictions.
  • Model Training: The model was trained in batches of 64 for up to 35 epochs, using validation data to monitor performance and early stopping to prevent overfitting. Model checkpointing preserved the best-performing weights for reliability. Real-time data augmentation techniques, such as rotations and flips, enhanced robustness and generalization. Despite challenges requiring week-long adjustments, the final Training-Validation Loss Trend highlighted strong performance, demonstrating the model’s effectiveness on unseen data.

Unit 11: Key Outcomes and Lessons

  • Outcomes: The model achieved 55.22% accuracy with a loss of 1.24, aligning with validation trends but falling short of state-of-the-art benchmarks (90-99%). Performance varied across classes, excelling in structured categories like ship (F1: 0.72) but struggling with complex ones like cat (F1: 0.32). The confusion matrix highlights strengths in frog (6) and ship (8) predictions but reveals challenges with visually similar categories like airplane vs. ship.
  • Recommendations: Performance could be improved by using larger datasets like CIFAR-100, optimizing hyperparameters such as batch size, learning rate, and epochs, and implementing advanced architectures or increasing convolutional filters for better class differentiation.
  • Lessons Learnt: I improved my Python programming skills, debugging techniques, and gained an appreciation for a slower, iterative process that prioritizes stability over complexity. Working directly in Python, despite being more difficult, provided valuable hands-on experience and deepened my understanding of Convolutional Neural Networks (CNNs), along with their training and optimization processes. Visualizing progress through graphs and metrics was helpful to check the progress and achieve a good first result.
